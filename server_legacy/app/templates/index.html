GymApp - App de Rutinas de Gimnasio

Este documento resume las características principales de la aplicación de rutinas y la base de datos integrada.

1. Propósito General

Desarrollar una aplicación que centralice una base de datos exhaustiva de ejercicios de musculación y fitness, integrando rutinas, seguimiento y visualización de imágenes anatómicas. El sistema debe permitir personalización y adaptarse a distintos objetivos.

  • Registro de usuario y perfil físico con datos opcionales como edad, peso y objetivos.
  • Los datos se usan para adaptar rutinas, análisis y recomendaciones.

2. Estructura de la Base de Datos (Excel)

Archivo fuente: BD_Ejercicios_mejorada_v8.xlsx

Columnas principales:

  • ID: Identificador único (ej. EX0001)
  • Nombre (ES): Nombre en español
  • Nombre (EN): Nombre en inglés
  • Zona principal: Músculo o grupo muscular principal
  • Músculos secundarios: Otros músculos involucrados
  • Detalle/Descripción: Guía técnica y consejos de ejecución
  • Imagen: Ruta del archivo (ej. /img/EX0001.jpg)

3. Gestión de Imágenes

  • Cada ejercicio cuenta con una imagen anatómica representativa ubicada en /img/.
  • Formato de nombre: [ID].jpg con tamaño estándar de 124 x 124 px.

4. Estructura y Funcionalidad de la App

a. Backend

  • Lectura de la base Excel y consultas filtradas.
  • Gestión de rutinas personalizadas y registro de progresos.

b. Frontend

  • Visualización de ejercicios y filtros por zona muscular, equipo o nivel.
  • Calendario de entrenamientos y carga rápida de progresos.

c. Funcionalidades Clave

  • Buscador de ejercicios.
  • Generador de rutina automática según objetivo y nivel.
  • Historial de seguimiento y análisis mensual.

5. Reglas y Buenas Prácticas

  • ID único e inalterable para cada ejercicio.
  • Actualización centralizada en el Excel.
  • Backup antes de cambios estructurales.
  • Esquema preparado para futuros campos opcionales.

6. Flujo de Trabajo Sugerido

  1. Carga y revisión del Excel.
  2. Mapeo de columnas y sincronización de imágenes.
  3. Desarrollo del frontend y pruebas.
  4. Iteración y mejoras continuas.

7. Observaciones Finales

  • El Excel es el núcleo del sistema.
  • Las imágenes deben estar correctamente nombradas y ubicadas.
  • El diseño facilita la consulta y el mantenimiento de la base de datos.
